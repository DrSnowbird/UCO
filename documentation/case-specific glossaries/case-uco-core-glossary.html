A
Annotation

A kind of Assertion. A valid occurrence satisfies the following necessary condition:

  • Ref at least one occurrence of UcoObject.

A statement asserted to be true in relation to one or more other objects.

B
Build

A kind of Facet. A valid occurrence may have the following properties:

  • buildInformation at most one occurrence of BuildInformationType.

(Undocumented)

Bundle

A kind of EnclosingCompilation and UcoObject.

(Undocumented)

C
Confidence

A kind of Facet. A valid occurrence satisfies the following necessary condition:

  • Confidence exactly one occurrence of ControlledVocabulary.

An asserted level of certainty in the accuracy of some information.

ControlledVocabulary

A kind of UcoObject. A valid occurrence satisfies the following necessary condition:

  • Value exactly one value of BasicString.

A valid occurrence may also have the following properties:

  • constrainingVocabularyName at most one value of BasicString.
  • constrainingVocabularyReference at most one value of URI.

A string value from an explicitly constrained set of string values.

E
ExecutionEnvironment

A kind of Facet. A valid occurrence satisfies the following necessary condition:

  • executionEnvironment exactly one occurrence of ExecutionEnvironmentType.

(Undocumented)

G
GPSCoordinates

A kind of Facet. A valid occurrence may have the following properties:

  • Hdop at most one value of double.
  • Pdop at most one value of double.
  • Tdop at most one value of double.
  • Vdop at most one value of double.

A GPS location, a possibly timed global position.

GranularMarking

A valid occurrence may have the following properties:

  • contentSelectors any number of values of BasicString.
  • markingRefs any number of occurrences of MarkingDefinition.

Marking definitions to be applied to particular portions of a particular UCO object.

I
Identity

A kind of UcoObject.

Characterization of the identifying properties of an individual or organization.

L
LatLongCoordinates

A kind of Facet. A valid occurrence may have the following properties:

  • Latitude at most one value of Decimal.
  • Longitude at most one value of Decimal.
  • Altitude at most one value of Decimal.

A geolocation expressed as latitude and longitude.

Location

A kind of UcoObject.

A geophysical place, site or position.

M
MarkingDefinition

A kind of UcoObject. A valid occurrence satisfies the following necessary condition:

  • definitionType exactly one value of BasicString.

A valid occurrence may also have the following properties:

  • Definition any number of occurrences of MarkingModel.

Represents a specific marking that may be applied to UCO data.

MarkingModel

A particular format of data markings

R
Relationship

A kind of UcoObject. A valid occurrence satisfies the following necessary conditions:

  • isDirectional exactly one value of Boolean.
  • targetRef exactly one occurrence of UcoObject.
  • sourceRef at least one occurrence of UcoObject.

A valid occurrence may also have the following properties:

  • startTime any number of values of Timestamp.
  • endTime any number of values of Timestamp.
  • kindOfRelationship at most one occurrence of ControlledVocabulary.
  • Confidence at most one occurrence of ControlledVocabulary.

An association or link between two uco objects.

Role

A kind of UcoObject.

Usual or customary function based on contextual perspective.

S
SimpleAddress

A kind of Facet. A valid occurrence may have the following properties:

  • Street at most one value of BasicString.
  • City at most one value of BasicString.
  • State at most one value of BasicString.
  • Zip at most one value of BasicString.
  • Country at most one value of BasicString.
  • addressType at most one value of BasicString.

A geolocation expressed as a simple administrative address.

T
Tool

A kind of NeutralRole and UcoObject. A valid occurrence may have the following properties:

  • toolType at most one value of BasicString.
  • Creator at most one value of BasicString.
  • Version at most one value of BasicString.
  • servicePack at most one value of BasicString.
  • References any number of values of URI.

Characteristics of a tool used in a cyber context potentially including its usage environment and configuration characteristics.

ToolConfigurationType

A kind of Facet. A valid occurrence may have the following properties:

  • configurationSettings any number of occurrences of ConfigurationSettingType.
  • Dependencies any number of occurrences of DependencyType.
  • usageContextAssumptions any number of values of StructuredText.

Characterizes the configuration for a tool.

U
UcoObject

A valid occurrence satisfies the following necessary condition:

  • ID exactly one value of Identifier.

A valid occurrence may also have the following properties:

  • Type any number of values of BasicString.
  • specVersion at most one occurrence of ControlledVocabulary.
  • createdTime at most one value of Timestamp.
  • createdByRef at most one occurrence of Identity.
  • objectMarkingRefs any number of values of Identifier.
  • Name at most one occurrence of ControlledVocabulary.
  • Description any number of values of StructuredText.
  • Facets any number of occurrences of Facet.
  • granularMarkings any number of occurrences of GranularMarking.

A cyber-relevant concept. This is the base object defining the minimal core set of properties to act as a consistent, unifying and interoperable foundation for all explicit and interrelateable content objects within the Uniform Cyber Ontology (uco).
