A
action

A kind of uco-descriptive-object. An instance may have the following properties:

  • action_status at most one instance of controlled-vocabulary.
  • action_start_time at most one value of timestamp.
  • action_end_time at most one value of timestamp.
  • errors any number of instances of action-error.
  • action_count at most one value of NonNegativeInteger.
  • facets any number of instances of facet.

Something that may be done or performed.

action-argument-facet

A kind of facet. A valid instance satisfies the following necessary conditions:

  • name exactly one instance of controlled-vocabulary.
  • value exactly one instance of controlled-vocabulary.

A grouping of properties characterizing a single parameter for an action.

action-error

A valid instance satisfies the following necessary conditions:

  • error_type exactly one instance of controlled-vocabulary.
  • error_content exactly one value of BasicString.

A characterization of the difference between the expected and the actual performance of an action.

action-estimation-facet

A kind of facet. An instance may have the following properties:

  • objective at most one value of BasicString.
  • estimated_impact at most one value of BasicString.
  • estimated_cost at most one value of BasicString.
  • estimated_efficacy at most one value of BasicString.

A grouping of properties characterizing decision-focused estimation aspects for an action that may potentially be performed.

action-frequency-facet

A valid instance satisfies the following necessary conditions:

  • rate exactly one value of Float.
  • units exactly one value of BasicString.
  • scale exactly one value of BasicString.

A grouping of properties characterizing the frequency of occurence for an action.

action-lifecycle

A kind of action. A valid instance satisfies the following necessary condition:

  • phase_refs at least one instance of action.

An action pattern consisting of an ordered set of multiple actions or sub action-lifecycles.

action-pattern

A kind of pattern. An instance may have the following properties:

  • pattern_expression at most one value of pattern-expression.

A logical pattern of characteristic action property values.

action-refs-facet

A kind of facet. An instance may have the following properties:

  • result_refs any number of instances of uco-object.
  • participant_refs any number of instances of uco-object.
  • instrument_refs any number of instances of uco-object.
  • location_refs any number of instances of location.
  • target_object_refs any number of instances of uco-object.
  • agent_ref at most one instance of uco-object.

A grouping of properties characterizing the core elements (who, how, with what, where, etc.) for an action.
